  <div className="mb-12">
    <h2 className="text-2xl sm:text-3xl font-semibold text-gray-900 dark:text-white mb-4">
      Quick Start
    </h2>
    <p className="text-base text-gray-600 dark:text-gray-400 mb-8">
      Get your mirror driver up and running in minutes
    </p>
  </div>

  <Steps>
    <Step title="Set up your GitHub repository">
      Create a new GitHub repository to host the mirror driver workflow. This repository will contain the GitHub Actions configuration that performs the mirroring.
    </Step>
    
    <Step title="Configure authentication">
      Create a GitHub personal access token with repository access and add it as a secret named `FRITES_2026_MIRROR_TOKEN` in your repository settings.
      
      <Note>
        The token needs `repo` scope to push to the target GitHub mirror repository.
      </Note>
    </Step>
    
    <Step title="Deploy the workflow">
      Add the workflow file to `.github/workflows/gitlab-mirror.yaml` in your repository:
      
      ```yaml .github/workflows/gitlab-mirror.yaml
      name: GitLab → GitHub Mirror

      on:
        schedule:
          - cron: "*/10 * * * *"
        workflow_dispatch:

      jobs:
        mirror:
          runs-on: ubuntu-latest
          steps:
            - name: Clone GitLab repo
              run: |
                git clone --mirror https://gitlab.com/ftc-civ/frites/2026.git repo.git

            - name: Push to GitHub mirror
              run: |
                cd repo.git
                git push --mirror https://x-access-token:${{ secrets.FRITES_2026_MIRROR_TOKEN }}@github.com/Thibault-Monnier/frites-2026.git
      ```
    </Step>
    
    <Step title="Verify synchronization">
      Navigate to the Actions tab in your GitHub repository to monitor workflow runs. The first sync will happen within 10 minutes, or you can trigger it manually using the workflow_dispatch option.
    </Step>
  </Steps>
